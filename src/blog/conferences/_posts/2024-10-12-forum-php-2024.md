## Create your own malware sandbox - Alexandre Daubois

Le contexte projet pour la conférence sera celui d'un pentest, le projet d'Alexandre ayant reçu un audit qui demande de mieux sécuriser l'upload de fichier.

3 options :
- Abandonner (pas possible pour le client)
- Une sandbox
- Créer leur propre antivirus

L'option de créer leur propre antivirus est avant tout envisagée, la détection heuristique étant explorée mais ayant trop de défauts pour être implémentée ici.

- Difficile a setup
- Faux positifs
- Faux négatifs

La solution suivante sera donc d'utiliser la signature de fichier.

<img src="/build/front/images/blog/sandbox.jpg">

### File signature check

Une courte introduction au chiffrement et au hashage nous permettra de voir que la solution nécessite soit d'utiliser un service externe connaissant les hash de fichiers verrolés, soit de l'entretenir soit même.

En considérant le fait que l'application est un back office interne, authentifié et privé, une sandbox maison semble être la meilleure solution pour répondre aux exigences du pentest.

Voici les grandes étapes :

1. Créer un répertoire de sandbox
2. Logger, reset de la sandbox.
3. On cherche les caractères interdits dans le nom du fichier. Par exemple, avoir une .php, ".." pour jouer dans l'arborescence ou autres caractères d'échappement.
4. Lecture du fichier, on change les permissions pour retirer les droits d'éxécution.
5. Les octets magiques : en lisant le binaire dump par PHP 8.2 sous macos, on voit quelques octets qui permettent de lire l'éxécutable.
6. On lis les magic bytes du fichier pour savoir de quel type est le fichier donné en sandbox, en cas d'éxécutable on le jette.

Un petit cas exceptionnel existe, les archives..

### Comment gérer les archives

Toutes les sécurités appliqués aux fichiers sont ici réappliqués à chaque fichier et dossier qu'on retrouve dans l'archive avec l'aide de la classe \PharData de PHP pour vérifier certaines attaques possibles.

- Tar bomb
- Récursion infinie
- Permission de fichiers dans les archives

Il aura fallu moins de 10 classes PHP pour créer cette sécurité.
Voila, la Sandboxed Directory Forbidden characters Magic Bytes with endianness et Archives handling est née. Si vous ne comprenez pas tout ces termes.. retrouvez la rediffusion bientôt sur la chaine de l'AFUP!

## Comment deboguer xdebug - Kevin Dunglas

Cette conférence est un REX sur la dernière fois ou Kevin s'est retrouvé bloqué devant du code ou une prod sans savoir quoi faire.

Kevin s'est retrouvé à avoir un soucis dans FrankenPHP, sa solution de serveur PHP à haute performance.

Grâce à un arbre de décision, il déduit si le bug vient d'une librarie ou du code projet (sans certitude en revanche).

<img src="/build/front/images/blog/kevin-graph-1.jpg">

Un autre arbre de décision a été fait, il est du point de vue du mainteneur.

<img src="/build/front/images/blog/kevin-graph-2.jpg">

Deux rapports de bug ont été fait sur dunglas/symfony-docker puis laravel/octane. Tous liés à FrankenPHP et Xdebug.

La nouvelle version de PHP mettra Kevin sur une fausse piste (mais si, celle du downgrade!).

Xdebug permet de faire de debug par étape, avoir de meilleures erreurs, du profiling ou encore du code coverage. Il est intégré à beaucoup d'IDE et dans de nombreuses stack PHP par défaut.

Petit clin d'oeil a Derick Rethans "Better debugging with xdebug" qui permet de mieux comprendre xdebug !

La piste se trouve autour de Xdebug et les builds ZTS, sans stacktrace il est difficile d'avoir des certitudes...

### Première étape : reproduire le soucis

Le bug n'étant pas reproduit en local, l'investigation était compliquée. Par chance un utilisateur va partager un exemple reproduisible !
```php
<?php

error_reporting(0);
sleep(1);
phpinfo();

// In the same directory, with xdebug enabled : frankenphp php-server
```

Mais le bug se produit de temps en temps (oui, de temps à temps..).

### Deuxième étape : récolter des informations

Si le code se trouvait dans le code PHP, on pourrait utiliser xdebug mais ici on doit débugger le débuggeur... Donc le bug se trouverait du coté du C ou autre.
Le GNU debugger permet de vérifier tout langage compilé, il sera donc utile ici !

Les programmes étant installés en mode production/optimisé, le debugger montrera des caractères illisibles pour l'humain car il manque les "Debug symbols" au lieu d'adresses mémoires..
Une solution simple se trouve de recompiler les sources des logiciels qui posent soucis, ici par exemple PHP avec le moins d'extension possible et si un bug se trouve dans la source, on peut tester son fix directement.

Une demo live du reproducer montrera la stacktrace et une backtrace qui indique dans son cas une récursion infinie, xdebug s'appelle lui même ce qui produit des soucis de CPU et le crash du programme.

En cherchant dans xdebug, xdebug remplace la fonction error_reporting puis remet la fonction d'origine ensuite. Ce comportement est documenté par PHP pour les extensions, malheureusement xdebug n'avait pas prévu le mode ZTS et donc remplace la fonction remplacée par la fonction remplacée etc... En faisant quelques modifs, le patch a pu se faire facilement et devenir thread safe.

Donc :
- Communiquer
- Obtenir des preuves
- Utiliser les bons outils
- Utiliser un moteur de recherche (ou lire des livres!)
- Développeur des théories
- Contribuer !

Ce talk nous aura permis d'avoir les outils pour debugger nos propres programmes efficacement et est un excellent REX étape par étape d'un bug complexe à détecter.

## Le cycle de vie d'une variable - Benoit VIGUIER

Benoit nous propose un rapide tour d’horizon des pièges et outils à connaître pour optimiser la gestion mémoire de nos programmes.

Composer dans leur commit ont utilisé la fonction "gc_disable()" afin de désactiver le collecteur de reference circulaire du gargabe collector de PHP ce qui a crée une émulsion dans la communauté PHP.

Ils ont pu justifier cet usage exceptionnel est dû au cas de longs scripts en console. Contrairement aux appels web qui sont destinés à peu durer, les appels CLI peuvent durer plus longtemps et mettre à mal le garbage collector de PHP et permettre des fuites mémoires.
